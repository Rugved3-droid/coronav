# CoroNav Baseline Results

**All numbers on this page are results as reported in the paper (original
runs)** — computed by the original research codebase's `_lm_metrics()`
during the paper's experiments, on the seeds frozen in
`benchmarks/protocol.json`. This repo's `coronav.metrics.lm_metrics()` is a
faithful reimplementation of the same metric definition, provided so you can
compute *comparable* numbers from your own runs — the table below is not
regenerated by this repo, it is the paper's own reported data.

**This page does not contain any numbers produced by running the Docker
container in this repo.** For the container's verification/reproduction
status, see the "Reproducibility status" section in `README.md` — it is
kept separate and does not appear here.

**Reference:** Parmar et al. 2026.
*VLM-guided coronary navigation: a benchmark for fluoroscopy-based guidewire
control.* (IEEE T-MRB, under review)

---

## Primary result: overlay effect (wire_only vs. overlay fluoroscopy)

Overlay fluoroscopy = vessel boundary visible in addition to guidewire.

| Anatomy | Condition | Median lm_rot_mm | n | Mann-Whitney U p | Effect direction |
|---------|-----------|-----------------|---|-----------------|-----------------|
| A1 (VMR0066) | wire_only | 0.1452 | 25 | — | — |
| A1 (VMR0066) | overlay | 0.1749 | 25 | **p = 0.0004** | overlay > wire_only |
| A2 (ASOCA N12) | wire_only | — | 25 | — | — |
| A2 (ASOCA N12) | overlay | — | 25 | **p = 0.000003** | overlay > wire_only |
| A3 (ASOCA N5) | wire_only | — | 25 | — | — |
| A3 (ASOCA N5) | overlay | — | 25 | **p = 0.000009** | overlay > wire_only |

All three anatomies show significant overlay > wire_only effect (all p < 0.001).
Cross-anatomy replication is the primary robustness evidence.

**A1 effect size:** r = 0.587 (large; Cohen's U3 convention).

### Interpretation

Higher `lm_rot_mm` in the overlay condition means the model makes *more* rotation
actions (relative to total wire movement) when the vessel boundary is visible.
This is consistent with the model using the visible vessel wall to guide directional
choices at the LM bifurcation.

## Prompt ablation (A1 only, n=10 per variant)

These results are from an **underpowered secondary check** (n=10 per variant vs.
n=25 required for 80% power at the observed effect size r=0.587).  They are
**not** the primary robustness evidence.

| Variant | Description | n per arm | wire_only median | overlay median | Δ | p (two-sided) | Interpretation |
|---------|-------------|-----------|-----------------|----------------|---|--------------|---------------|
| V1 | Full scaffold (DEFAULT_SYSTEM_PROMPT) | 25 | 0.1452 | 0.1749 | +0.0297 | **0.0004** | SIGNIFICANT |
| V2 | No spatial scaffold | 10 | 0.0000 | 0.0000 | — | 0.1681 | UNINFORMATIVE (both arms ~zero rotation) |
| V3 | Compressed / minimal | 10 | 0.0970 | 0.0428 | −0.0542 | 0.1375 | EXCLUDED (navigation collapsed; reversed direction) |
| V4 | Rephrased scaffold | 10 | 0.1398 | 0.1476 | +0.0078 | 0.2413 | Same direction, NS (n too small) |

V2 is uninformative: both arms had near-zero LM rotation, meaning the operator
barely moved during the LM phase under either condition.

V3 is excluded: navigation collapsed under the compressed prompt (0/10 success
in both arms); the reversed sign reflects degenerate behaviour, not a true reversal
of the overlay effect.

V4 is consistent in direction with V1 but underpowered.

## Success rate, steps-to-target, and branch selection (A1, n=25 per arm)

From Table I of the paper.

| Metric | wire_only | overlay | Test (two-sided) |
|--------|-----------|---------|-------------------|
| Episodes reaching mid-LAD (success) | 20 / 25 (80%) | 13 / 25 (52%) | Fisher p = 0.072 (n.s.) |
| Median steps to target (successes only) | 48.0 | 58.0 | U = 24, p = 0.0001, r = 0.812 |
| Steps to target, seed-matched (n=9 both succeeded) | — | — | Wilcoxon W = 3, p = 0.020 |
| First-bifurcation LCx (wrong branch) | 8 / 25 | 9 / 25 | Fisher p = 1.000 (n.s.) |

The success-rate difference is directional but **not statistically
significant** (Fisher p = 0.072) — do not report it as a significant
finding on its own. The significant, pre-specified result is the
steps-to-target comparison among successful episodes: seed-matched Wilcoxon
signed-rank on the 9 seeds where both arms succeeded (overlay slower on 8 of
9 matched seeds). First-bifurcation branch selection shows no difference
between conditions (NULL result) — an n=10 subsample suggested an effect
(p=0.033) but it did not survive resampling to n=25; report only the n=25
NULL result.

## Run the benchmark yourself

This runs the same protocol (seeds, step budgets, metric definition) as the
paper. **It has not been validated to reproduce the numbers above at full
scale (n=25)** — see "Reproducibility status" in `README.md` for the
current, partial (n=9) smoke-run result and its gap from the paper's median.
Treat any numbers you get as your own run's result, not a confirmed match
to this page, until a full n=25 validation is done.

```bash
# A1 (bundled — no registration)
python benchmarks/run_benchmark.py --operator claude --anatomy a1

# View results
python benchmarks/report.py --run-dir runs/<timestamp>/
```

For A2/A3, first run `python scripts/download_asoca.py` after registering at
https://asoca.grand-challenge.org/

## Submit your results

If you benchmark a new operator and want to add it to this table, open a PR with:

- [ ] `trace.jsonl` for each anatomy (or a link to a public data deposit)
- [ ] `scripts/check_install.py` output confirming SOFA version and anatomy hash
- [ ] Operator description (model, API version, any prompt changes)
- [ ] Whether you used the default `DEFAULT_SYSTEM_PROMPT` or a custom prompt

We will recompute `lm_rot_mm` using `coronav.metrics.lm_metrics()` from your trace.

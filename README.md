# CoroNav

**Open benchmark for VLM-guided coronary guidewire navigation via fluoroscopy.**

CoroNav evaluates multimodal language models as real-time operators in a
physics-based coronary artery simulation (SOFA v23.06 + BeamAdapter).  The
model sees a fluoroscopy image and a one-line state string, and must steer a
guidewire to the mid-LAD target by issuing translation/rotation commands.

```
fluoroscopy image (512×512 px)   state text: "Distance to target: 42.3 mm. Trend: decreasing."
       │                                  │
       └──────────────┬───────────────────┘
                      ▼
             VLM operator (your model)
                      │
          {"intended_direction": "advance",
           "action": {"translation": 30, "rotation": 0}}
                      │
                      ▼
          SOFA v23.06 coronary simulation
```



---

## Reproducibility status

- **Paper results** (`RESULTS.md`) are the original runs from the paper's
  research codebase, reported as-is. They are not regenerated by anything in
  this repo.
- **This Docker container** builds and runs the full CoroNav pipeline
  end-to-end (SOFA v23.06 + BeamAdapter + stEVE + CoroNav) and lets you run
  your own model against the same environment.
- **Full-scale container reproduction of the paper's baseline (n=25) has not
  been validated.** A partial, small-sample smoke-run is documented below —
  it is provided to show the pipeline executes correctly, not as evidence
  the container reproduces the paper's numbers. Treat this as a known, open
  item, not a finished validation.

## Install

**SOFA v23.06 + BeamAdapter are compiled C++ binaries — they cannot be
pip-installed.**  Docker is the supported install path.  The image downloads
two pre-built Linux binaries (~338 MB) and installs everything automatically.

### Prerequisites

- Docker (any version with `docker build`)
- An Anthropic API key (`ANTHROPIC_API_KEY`)

### Build the image (~5 min, downloaded layers are cached)

```bash
git clone https://github.com/Rugved3-droid/coronav.git
cd coronav
docker build -t coronav:latest -f docker/Dockerfile .
```

The build runs `python3 scripts/check_install.py` as the final layer — a
passing build means SOFA, BeamAdapter, stEVE, and CoroNav all imported
successfully inside the container, and that the A1 centerline data is
present *and loadable* (not just present — `check_install.py` actually
parses it via `left_coronary_branches()`).

**Verification status:** this Dockerfile builds successfully and runs the
full pipeline end-to-end (SOFA v23.06 + BeamAdapter + stEVE + CoroNav) with
real VLM-driven navigation — imports, physics, and rendering all confirmed
working. **Full-scale reproduction of the paper's baseline has not been
validated** — see "Container smoke-run" below and "Reproducibility status"
above. Do not assume this container reproduces the paper's 0.145 median.

### Run the quickstart (A1, 3 episodes, ~5–15 min)

> **Windows / Docker Desktop users:** the `-v $PWD/runs:/coronav/runs` mount
> below only works if the drive you're running from is enabled under
> Docker Desktop → Settings → Resources → File sharing. By default this is
> often just `C:`. If your clone lives on another drive (`D:`, etc.) and
> that drive isn't shared, the mount silently no-ops — the container starts
> fine, produces no error, and any output written to `runs/` simply never
> appears on the host. Enable your drive in File sharing (Apply & Restart)
> before relying on mounted output.

```bash
docker run --rm -it \
  -e ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY \
  -v $PWD/runs:/coronav/runs \
  coronav:latest \
  python3 examples/quickstart.py
```

Expected output:
```
=== CoroNav Quickstart ===
Anatomy: A1 (VMR0066) at /coronav/data/anatomy/a1_vmr0066
[INFO] [PluginManager] Loaded plugin: .../lib/libBeamAdapter.so
--- Episode 1/3  (seed=5001) ---
    step  1  dist=85.1->85.3  branch=left_main  advance  (...)
    ...
  lm_rot_mm = 0.XXXX  |  success = False  |  steps = None
=== Run Summary ===
  lm_rot_mm  median=0.XXXX  mean=0.XXXX
Paper baseline (n=25): wire_only median=0.1452 | overlay median=0.1749 | p=0.0004
```

> The wire reaches `left_main` and begins navigating.  Success=False in 3
> episodes is expected — the 60-step budget is tight and `wire_only` mode
> (no vessel overlay) is the harder condition.  The paper's baseline used
> n=25 episodes with the full benchmark seeds.

### Container smoke-run (partial — not a reproduction of the paper's results)

`coronav/prompts.py`, `coronav/operator/claude.py`, and the episode-runner
logic in `coronav/env.py` are ported from the paper's `run_coronary.py`
harness (staged system prompt with action-semantics grounding, hard-enforced
retraction budget and proximity gate, no-progress rotation handling), so the
container runs the same operator design as the paper.

**This has been exercised with a small, partial run — n=9, not the paper's
full n=25 — to confirm the pipeline executes correctly end-to-end. It is
not presented as a reproduction of the paper's baseline:**

| | n | median `lm_rot_mm` | success |
|---|---|---|---|
| Paper (`RESULTS.md`, n=25, original runs) | 25 | 0.1452 | 20/25 (80%) |
| Container smoke-run (this repo, n=9*) | 9 | 0.1911 | 6/9 (66.7%) |

\* Seeds 5001–5100 (paper's own documented seeds), `wire_only` mode. One
episode (seed 5100) is excluded — the API key used for this test ran out of
credits mid-episode, producing zero valid API responses for that episode.

The container's numbers are **not a numeric match** to the paper's — off by
about 32% at n=9, a sample too small to draw a reproduction conclusion from
either way. The individual episode values do fall within the paper's own
observed per-episode spread (0.105–0.371 at n=25), and this is a large
improvement over an earlier, broken port of the operator (which scored
0.29–0.92 median with 0% success before a rotation-lock bug was fixed) — but
neither of those facts substitutes for an actual n=25 reproduction run,
which has not been performed. **Full-scale container reproduction of the
paper's baseline is an open item**, not a validated result. If you run one,
please open a PR with your trace files.

## Run the full benchmark

```bash
# A1 (bundled anatomy, no registration required)
docker run --rm -it \
  -e ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY \
  -v $PWD/runs:/coronav/runs \
  coronav:latest \
  python3 benchmarks/run_benchmark.py --operator claude --anatomy a1

# View results
docker run --rm \
  -v $PWD/runs:/coronav/runs \
  coronav:latest \
  python3 benchmarks/report.py --run-dir runs/<timestamp>/
```

For A2 + A3 (ASOCA, requires free registration):

```bash
# 1. Register at https://asoca.grand-challenge.org/
# 2. Download the ASOCA Normal zip to your host machine
# 3. Install into the mounted anatomy volume:
docker run --rm \
  -v $PWD/data/anatomy:/coronav/data/anatomy \
  -v /path/to/ASOCA_Normal.zip:/tmp/ASOCA_Normal.zip \
  coronav:latest \
  python3 scripts/download_asoca.py --asoca-zip /tmp/ASOCA_Normal.zip

# 4. Run all three anatomies
docker run --rm -it \
  -e ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY \
  -v $PWD/runs:/coronav/runs \
  -v $PWD/data/anatomy:/coronav/data/anatomy \
  coronav:latest \
  python3 benchmarks/run_benchmark.py --operator claude --anatomy a1 a2 a3
```

## Add your model

Copy `coronav/operator/template.py`, rename the class, fill in `act()`.
You only need to implement one method — everything else is handled by the
benchmark runner.

```python
from coronav.operator.base import VLMOperator
from coronav.prompts import DEFAULT_SYSTEM_PROMPT

class MyOperator(VLMOperator):
    def __init__(self):
        super().__init__(DEFAULT_SYSTEM_PROMPT)
        # load your model / client here

    def act(self, images, state_text):
        # images: list of (512,512) uint8 arrays
        # state_text: "Distance to target: 42.3 mm. Trend: decreasing. ..."
        # return: {"intended_direction": "advance", "action": {"translation": 30, "rotation": 0}}
        ...
```

See `examples/add_your_model.py` for a complete GPT-4o implementation.
Then benchmark from inside the container:

```bash
docker run --rm -it \
  -e OPENAI_API_KEY=$OPENAI_API_KEY \
  -v $PWD/runs:/coronav/runs \
  -v $PWD/coronav/operator:/coronav/coronav/operator \
  coronav:latest \
  python3 benchmarks/run_benchmark.py \
    --operator coronav.operator.my_model.MyOperator \
    --anatomy a1
```

## Advanced: running without Docker

> **Warning:** Building SOFA v23.06 + BeamAdapter from source typically takes
> several hours and requires specific system libraries.  Docker is strongly
> preferred.  This path is for users with an existing SOFA v23.06 + BeamAdapter
> installation (e.g., the paper authors' Windows setup).

If you already have SOFA v23.06 and BeamAdapter installed:

```bash
# Set SOFA paths
export SOFA_ROOT=/path/to/SOFA_v23.06.00_Linux
export PYTHONPATH="${SOFA_ROOT}/plugins/SofaPython3/lib/python3/site-packages:$PYTHONPATH"
export LD_LIBRARY_PATH="${SOFA_ROOT}/lib:$LD_LIBRARY_PATH"

# Install Python deps + stEVE + CoroNav
pip install numpy scipy Pillow anthropic
pip install git+https://github.com/lkarstensen/stEVE.git
pip install -e .

# Verify
python scripts/check_install.py

# Run
ANTHROPIC_API_KEY=... python examples/quickstart.py
```

## Metric

**Primary: `lm_rot_mm`** — fraction of wire-movement distance during the left
main (LM) phase that occurs on rotation steps.  Dimensionless [0, 1].

See `PROTOCOL.md` for full definition.  See `RESULTS.md` for paper baseline
numbers and instructions for submitting your results.
